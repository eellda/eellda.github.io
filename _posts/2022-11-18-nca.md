# NCA

클라우드의 기반 = 가상화  

## 사용 이유
- 비용 절감
- 셀프 서비스
- 종량제
- 보안
CAPEX - 자본적 지출 (투자) < OPEX - 운영 비용

## 서비스
1. 인프라 서비스
2. 플랫폼 서비스
3. 어플리케이션 서비스

### Region
국가.
- vpc, object storage 등이 region에 종속적

### Zone
IDC 로 이해하면 편함 / 네트워크 기준으로 카운트
- server, nas 등이 zone에 종속적

### compute 상품
서버를 생성하고 관리하는 상품
- 스토리지 비용 발생
- IO 퍼포먼스 차별화

### iops
SSD 구성시 ~100GB = max iops 4000 고정  
500GB~ = max iops 20000 고정
- 100GB 당 IOSP 4000 증가

### Bare Metal Server
- 하이퍼 바이저 없이 바로 OS가 올라가는 서버
- 서버 스펙이 매우 좋기 때문에 가능
- RAID 구성 방식 선택 가능
- 오라클 리눅스를 구성 가능
- 복구따위 하지 않음

### 서버 operation
1. CLI
텍스트 기반 UI.
컴퓨터에 있는 파일을 관리하고 볼 때 사용
2. API
두 응용 프로그램 간의 통신을 돕는 중개 소프트웨어
누가 만들걸 요청해서 가져다씀

### Srver Image
- window - 100GB
- linux - 50G
기본 제공

### init script
- 서버 생성할 때 실행되는 스크립트를 지정
- 패스워드 초기화 된걸 다시 한번 설정 가능
- NAS 마운트

### ACG
- 서버를 보호해주는 방화벽
- TCP, UDP, ICMP중 선택
- 화이트리스트 방식과 블랙리스트 방식의 방화벽

### 추가 스토리지
- 10G~2TB
- 15개의 추가 스토리지
- 리눅스는 디스크 묶을 수 있음 (볼륨매니저)
- 디스크를 다른 서버로 옮길 수 있음
- 디스크 확장 가능

### Kubernetes Service
컨테이터를 관리, 배포, 조정 해주는 역할

### Networking
DNS, CDN, VPC 등의 서비스를 제공

### VPC (Virtual Private Network)
- 가상의 레이어를 만들어 혼자 있는 것처럼 해주는 것
- 가상 네트워크
- 계정당 생성 가능
- 계정당 최대 3개

### VPC - Subnet
- 퍼블릭 서브넷은 서버만 위치 시킬 수 있고 공인IP 부여가능
- VPC마다 생성되고 타 VPC와 중복 안됨
- 가상 네트워크의 서브넷
- 프리베이트 서브넷은 서버 혹은 로드 밸런서를 위치 시킴

### VPN
- 서브넷으로 망끼리 연결 해줌

### Peering
VPC간 연결을 위한 네트워크 구성임

### Load balancer
L1 - hub
L2 - switch, MACadress 유무로 분기
L3 - 라우터 IP로 분기
L4 - 포트 (TCP, UDP) 로 보고 분기

### Load balancing 알고리즘
- round robin = 서버에서 1개씩 분배
- last connection = 연결이 제일 적은 서버에 새로운 커넥션을 분배
- s*ource IP hash = IP 를 함수 f(X)에 대입*
- *분배 알고리즘 3개*

### 프록시 
관리자라고 보면 됨 송수신 처리 -> SSL 도입 가능 처리 수 확인 가능

### DNS - 도메인 등록 서비스
ip에 이름을 매긴것
- 다양한 레코드 타입 지원 ( A, NS, SOA, CAA, TXT 등)

### CDN - Contents Delivery Network
- youtube, neflex가 거의 압도적
- 보다 빠르고 안정적으로 컨텐츠를 전송하는 서비스
- 메인 서버의 요청을 분산시켜주는 것이라고 보면 됨
- OBT, 동영상 서비스 등에 사용함

### Object Stroage
리소스 메니지먼트가 중요
비연속 할당 방식의 링크 구조
- 오브젝트가 하나의 단위(블럭이 아님)
- 파일 단위로 복제만 함
- 편집용도 X
- 비정형 데이터를 주로 사용

### 분산 파일 시스템
저장 공간을 하나로 묶은것

### Archive storage - Data Lifecycle
- 일정 기간이 지나거나 특정 조건이 될 경우 아카이브 스토리지로 옮겨짐
- 데이터 처리 비용은 비쌈

최근, 인기 - object storage
구형, 비인기 - archive storage

### NAS
다수의 서버에서 공유하여 사용할 수 있는 storage
- 500GB ~ 10TB, 100단위
- 자동 스냅샷 기능 ( 스냅샷 찍기 전에 삭제시 적용안됨)
- NFS / CIFS

### Data Teleporter

대용량 데이터(최대 100TB) 이전을 위한 어플라이언스 대여 서비스
